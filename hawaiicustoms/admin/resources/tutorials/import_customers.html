Importing Customers: A Quick Tutorial

If you have a list of customers in a file on your computer, you can upload that file in to the system and have the customers automatically added to your store. To start, you will need to have your list of customers in a CSV (comma separated value) formatted file. A CSV file will contain a list of your customer details in a line-by-line format, such as:

John,Smith,john@example.com,password,123 Example St
Jane,Smith,jane@example.com,password2,456 Example St

Move your mouse over the "Customers" tab and click the "Import Customers" option, as shown below:


To import customers from a file, click the "Import Customers" option under the "Customers" tab.

This will start the "Import Customers" wizard.

Step 1


Set the import settings and select the file to import from.

Import Details
  1. Override Existing Details?

    If you'd like to override the customer information of any existing customer with the same email address, check "Yes, override existing records."

    If left unchecked, any customers in your CSV file with an excisting matching email address in the store will not be imported.

File Details
  1. Import File

    To upload a file from your computer, select "Upload a file from my computer" and click "Browse..." to locate the file on your computer.

    Note: If the file size is larger than the noted maximum size limit, you will need to upload the file file via FTP in to the admin/import/ directory.

    If this is the case, select "Use a file already on the server" and select the file you uploaded.

  2. Contains Headers?

    If this CSV file contains a row of headers at the top before the actual data starts, check this option.

    A CSV file with a row of headers will look similar to the following:

    FIRST NAME,LAST NAME,EMAIL ADDRESS,PASSWORD,ADDRESS 1
    John,Smith,john@example.com,password,123 Example St
    Jane,Smith,jane@example.com,password2,456 Example St
  3. Field Separator

    If the fields in your CSV file are not separated by a comma (,), change this field to match your field separator.

    Note: If unsure, leave this setting as the default.

  4. Field Enclosure

    If the fields in your CSV file are enclosed within a different character (by default "), change this field to match your field enclosure.

    Note: If unsure, leave this setting as the default.

Click "Next >>" when you are done.

The upload process may take a moment or two to complete and parse your CSV file.

Step 2

You will now be asked to map the fields from the imported file to fields in the system.

Simply click the drop down box next to each of the system fields to select an appropriate field from your CSV file.


Click the drop down next to each field to map it to an appropriate field in the CSV file.

Note: If you do not specify a password field, the system will generate a random password for the user. The user can then reset their password using the lost password functionality.

Click "Next >>" when you are done.

Step 3

Click "Start Import" to begin the import process.

Your customers will be imported one by one and the popup window that appears will show you how many records have been imported and how many remain.

After the import process is complete you will see a final report which shows the status of the import as well as any incomplete or invalid records.


The final screen will show you how many customers were imported. It will also tell you if the import file contained bad data.

Appendix: Customer CSV Fields

The system supports numerous fields for importing customers from a CSV file.

  • Email Address: (String) The email address of the customer.
  • Password: (String)A plain text password for the customer. If not specified, a random password will be generated.
  • First Name: (String) The first name of the customer.
  • Last Name: (String) The last name of the customer.
  • Company: (String) The company of the customer.
  • Phone: (String) The phone number of the customer.
  • Shipping Name: (String) If a shipping address is to be created, the shipping address name. If not specified, the first & last names of the customer are used.
  • Shipping Address 1: (String) The shipping address for the customer. If not specified, a shipping address will not be created.
  • Shipping Address 2: (String) The shipping address line 2.
  • Shipping City: (String) The shipping address city.
  • Shipping Zip: (String) The shipping address zip or post code.
  • Shipping State: (String) The shipping address state.
  • Shipping Country: (String) The shipping address country.
  • Shipping Phone: (String) The phone number for this shipping address. If not specified, the customer phone number will be used.
